## 1. Схема базы данных
Для реализации проекта выбрана документоориентированная СУБД **MongoDB**. 
База данных `marathon_db` состоит из двух основных коллекций:
* **`runners`** (Бегуны): Хранит профили марафонцев.
  * Поля: `_id`, `runner_id` (UUID), `name` (Имя), `target_distance_km` (Целевая дистанция).
* **`workouts`** (Тренировки): Хранит логи пробежек.
  * Поля: `_id`, `runner_id` (ссылка на бегуна), `date` (Дата), `distance_km` (Дистанция), `avg_heart_rate` (Средний пульс).

## 2. Реализация шардинга
Для обеспечения горизонтального масштабирования был поднят кластер MongoDB с помощью Docker Compose, состоящий из:
* 1 Config Server (`configsvr`)
* 2 Shard Servers (`shard1`, `shard2`)
* 1 Router (`mongos`)

**Обоснование выбора ключа шардирования:**
Коллекция `workouts` шардирована по полю **`runner_id` с использованием хэшированного ключа (Hashed Sharding)** (`{ "runner_id": "hashed" }`). 
*Почему именно так?* В системах трекинга активности данные генерируются постоянно. Если использовать Range-based шардинг (например, по дате), все новые записи будут лететь в один шард, создавая "горячую точку" (hotspot). Хэшированный ключ по ID бегуна гарантирует равномерное распределение нагрузки и данных по всем доступным шардам (в нашем случае ~50/50 между shard1 и shard2), независимо от того, насколько часто бегают спортсмены.

## 3. Нагрузочное тестирование
Нагрузочное тестирование реализовано с помощью кастомного скрипта на Python. 
Вместо YCSB был написан генератор синтетических данных (с использованием библиотеки `Faker`), который имитирует массовую запись тренировок в базу данных через роутер `mongos`.

**Методология:**
* Использовался пул потоков (`concurrent.futures.ThreadPoolExecutor` на 4 воркера) для имитации параллельных запросов от разных пользователей.
* Производились замеры времени на вставку пакетов данных (Batches): 1 000, 5 000, 10 000 и 20 000 записей.
* Результаты выполнения скрипта логируются, а также автоматически визуализируются с помощью `matplotlib`.

## 4. Инструкция по запуску
1. Поднять кластер: `docker-compose up -d`
2. Инициализировать шарды через `mongosh`.
3. Установить зависимости: `pip install pymongo faker matplotlib`
4. Запустить интерфейс и тесты: `python app.py`
